# 为什么 eval2 至今无法超越：代码考古 + 文献调研与 alternative 评估

> 日期：2026-07-19
> 问题：「eval2 就是很久以前随便写的一个函数，做了这么久仍然没有超越，怎么这么巧？」
> 结论：**不是巧合。eval2 不是启发式，而是手牌效率子问题的精确 2 步随机动态规划；
> 我们一直在用一个有噪声的近似去打一个精确解的主场。** 文献上也没有人在同类岗位
> 赢过这类函数——顶级麻将 AI 全部绕开了这个岗位（不玩这个规则，或不打这个岗位）。

---

## 1. 代码考古：eval2 到底是什么

`algo/eval/legacy.py`：

```
eval0(hand)  = eval_naive(hand).metric()        # 精确面子/对子分解得分
eval1(hand)  = E_draw[ eval0(hand+1) ]           # 对剩余牌分布的 1 步精确期望
eval2(hand)  = E_draw[ eval1(hand+1) ]           # 2 步精确期望（eval_rec 两层）
```

- `eval_naive`（legacy.py:283）把全副手牌按花色切分后做**精确组合分解**
  （面子数 melds_num + 将 pairs_num，metric = melds + pair_coef×min(pairs,1)），
  带缓存穷举所有拆分取最优——不是拍脑袋权重，是组合最优解。
- `eval_rec` 的 2 步期望对**真实剩余牌分布**精确求和（不是采样）。
- 所以 eval2 = 在「只看自己手牌、目标=尽快和牌」这个子问题上的**精确 2 步
  随机 DP**。它唯一的"随便"成分是终端 eval0 的离散结构分，而 2 步 DP 的
  期望结构把这个分的噪声磨掉了大半。

附：`algo/eval/v2.py`（V3 系的 eval0 叶）是同族变体：精确 shanten（递归最优）
+ 搭子质量 + 待牌数。同样：核心量是**精确可算**的。

## 2. 为什么超越不了：岗位分析

「超过 eval2」意味着在手牌效率评估这个岗位上，用带噪监督学出的函数去逼近一个
**精确可计算的量**。NN 输的不是能力，是岗位性质：

- 象棋/将棋的手工评估子（子力+位置表）能被 NNUE 超越，因为那些局面评估
  （王安全、兵型、子力协调）**没有精确解**，手工式只是粗糙近似，headroom 巨大；
  且 NNUE 用**搜索标注**（深搜分数）而非裸结果训练。
- 麻将的向听/进张/两摸效率**精确可算**（组合 DP），eval2 已经接近这个子问题的
  精确解，headroom ≈ 0。NN 在这个岗位上最多逼近 eval2，永远只能以内。
- 麻将里**没有**精确解的部分（对手建模、防守、攻守判断）根本不在 eval2 的
  岗位描述里——它们在 BeliefExp 的 belief 层。我们的实测：隐藏信息全知只值
  +1.2%（方向 0），belief 特征 AUC 0.638（§7.14）——这些岗位的增量本来就小。

本项目自己的失败链（全部记录在案）：

| 尝试 | 结果 | 出处 |
|---|---|---|
| Deep Value Net / MC value 当叶 | 远逊 eval0，纯 NN leaf 全灭（最佳 −165 Elo） | td-lambda-plan §9.9 |
| residual leaf（eval0 + 2·nn） | TD value 语义冲突，−70 Elo 起 | td-lambda-plan §9.7 |
| JAXG value 头当叶（arena 直接 A/B） | ×100 平、×300 −4.6% 显著 | gumbel-deploy-0719 §1 |
| π' 部署（1-ply NN 搜索替代） | pool 0.5%，单调有害 | gumbel-deploy-0719 §2 |
| depth-2 教师蒸馏 | 80 局否决（**功率不足**，见 §5-C1） | fable-5-review §二.6 |

## 3. 文献对照：顶级麻将 AI 的 eval 岗位长什么样

| 系统 | 规则 | eval 岗位 | 有无手工评估函数 |
|---|---|---|---|
| [Suphx (MSRA)](https://arxiv.org/abs/2003.13590) | 立直 | 纯 NN（CNN）+ 全局奖励预测 + oracle guiding + 运行时 MC 策略适应 | **无**（立直的役/符/宝牌计分使手工评估组合爆炸，不可行） |
| [LuckyJ / 绝艺 (Tencent)](https://sports.sina.cn/others/qipai/2023-07-12/detail-imzamafw0364307.d.html) | 立直 | 纯自对弈 RL + 博弈论，不用人类牌谱（[对比分析](https://modern-jan.com/blog/luckyj_vs_naga_and_suphx/)） | **无** |
| NAGA (Dwango) | 立直 | 人类凤凰桌牌谱监督 + NN | **无** |
| Mortal | 立直 | offline RL 纯前馈 | **无** |

关键观察：**没有一家在「手工叶值函数」岗位上竞争**——原因不是手工函数弱，
而是他们打的立直麻将计分规则（役种×符×宝牌×场况）让手工评估在组合上不可行，
NN 是他们唯一的选择。我们的推倒胡计分极简（和/铳/流局），手工 2 步 DP 恰好
可行且近优——**eval2 是这个简单规则的礼物，不是历史的包袱**。

另一个对照点：Suphx 的运行时机制是 parametric Monte-Carlo policy adaptation
（在线微调策略），LuckyJ 全靠离线训练——顶级系统也都不做「手工叶值 + 深搜索」，
他们的强度来自训练侧，与我们的「训练侧 1b 搜索蒸馏 + 部署侧 Hybrid」结构判断一致。

## 4. 「怎么这么巧」的直接回答

1. eval2 是精确 DP，不是随便写的启发式（结构精确，只有终端权重是手调的）。
2. 我们三年里尝试替代它的岗位（叶值/1-ply 搜索值/部署搜索）全部是它的主场。
3. 它打不了的那份工（对手建模、防守），BeliefExp 的 belief 层在做，实测增量也小。
4. 文献上没有人在同类岗位赢过这类函数；立直系 AI 绕开这个岗位是因为他们的
   规则让手工评估不可行。

## 5. Alternative 清单（逐一评估）

### C1. 更深精确 DP / 功率充分的深教师重测 —— 唯一有真实 headroom 的方向

**已测（2026-07-19，1000-pair duplicate，功率充分重审当年 80 局否决）**：
`v3deep:2-eval0-eval0` vs `v3deep:1-eval0-eval0`（同骨架同候选同叶，仅 DP 深度不同），
结果 **+1.2% [−1.0,+3.4]**，score-proxy +0.006 [−0.047,+0.059]，95% 不显著；
同池 BeliefExp 20.6% ≈ depth-2 的 21.1%，均远低于 Hybrid-Base 35.1%。
**判定：eval2 家族深度已饱和**——更深 DP 的边际收益 ≤1% 且已被现役 BeliefExp
的深度收割；当年 80 局否决的结论在充分功率下成立（方向不变、量级更小）。
pkl：`output/ab_depth2_vs_depth1_1000.pkl`。

### C2. MC 手牌完成价值（期望和牌巡数/和牌率，剩余牌条件）

eval0 语义的直接升级版：用模拟估计「这手牌 X 巡内和牌的概率」当叶。
比 shanten/ukeire 信息更全，但 ukeire 已经是它的一阶近似，
且 algo-proposals 方案 B 当年结论「leaf 太粗糙前加深搜索放大误差」反向说明
leaf 精细化收益有限。预期：小。成本：中（要造快速模拟器）。

### C3. 学习 eval0 终端，保留 2 步 DP 结构

只替换 eval_naive 的结构分（最小侵入），用 eval2 自身的闭包值或搜索结果蒸馏
一个 NN 终端。理论上能学到 eval0 没编码的上下文（剩余牌、巡目）。
但 DP 结构不变 → 收益被终端 headroom 限制；eval0 的结构分已近组合最优。
预期：很小。成本：小。可作为 C1 的附属实验顺手做。

### C4. NNUE 式 search-labeled 叶（深搜标注训练叶网）

象棋 NNUE 的成功路径。但：教师 = eval2 系搜索 → NN 叶上界被教师锁死，
且部署时 winner's curse（TD leaf 实验已实证 −90 Elo 起）。
除非教师显著更强（= C1），否则无解。**与 C1 绑定，不独立成立。**

### C5. 承认 eval2 是已解决的子问题，强度增量换岗位

- 修 arena 副露和牌判负（is_succ quirk）：对齐训练环/真实规则的 meta 偏移，
  今天已量化（0.229% 漏判，全部副露和）；
- 引擎接入报听收益后重估各 agent 的报听行为（§7.11/7.12 的联动死代码会激活）；
- 这些不修 eval2，但改变 eval2 所处博弈的真实回报结构。

## 6. 推荐

1. ~~C1 优先~~ **C1 已结案（2026-07-19）**：depth-2 vs depth-1 = +1.2% [−1.0,+3.4]
   不显著，深度饱和（见 §5-C1）。eval2 的不可替代性最终确认：精确 DP 结构 +
   已被现役 BeliefExp 利用充分的深度。
2. C5 的规则修复是平台级正事（修完规则所有结论重测）。
3. C2/C3/C4 预期收益不支持单独立项；C4 已无成立前提。
