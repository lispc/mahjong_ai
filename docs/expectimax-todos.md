# Expectimax 框架下的改进 TODO

> 本文档记录在当前“多步前向期望搜索”算法骨架不变的前提下，晋北麻将 AI 可以逐步落地的改进项。
> 当前核心流程：`select()` → `eval_rec()` → `metric = eval_naive().metric()`，本质是用前向摸牌期望来选手牌。

---

## 一、评估函数（eval0）升级

当前评估仅使用：
```python
metric = 面子数 + pair_coef * min(对子数, 1)   # pair_coef = 0.6
```

改进方向：

- [ ] **1.1 引入标准向听数**
  - 计算手牌变成胡牌形最少需要几次有效进张。
  - 用拆牌 DP 替换当前粗略的“面子数 + 对子数”。
  - metric 可改为 `4 - 向听数` 或 `-向听数`。

- [ ] **1.2 听牌质量量化**
  - 对每种拆分结果，计算还能胡哪些牌、共多少张。
  - 加入奖励项：`γ * 听牌张数`。

- [ ] **1.3 搭子与浮牌质量分级**
  - 两面搭子（如 5、6 万）> 坎张（如 5、7 万）> 边张（如 1、2 万）> 孤张。
  - 中间张（4、5、6）潜力高于边张（1、2、8、9）。

- [ ] **1.4 字牌与幺九特殊权重**
  - 晋北推倒胡无役牌加分，单张字牌若无对子/刻子倾向，应降低保留价值。
  - 已有人碰出的字牌，该字牌成对概率归零。

## 二、概率模型（tile_prob）升级

当前 `Context.tile_prob()` 仅做均匀分布，且 `used` 字段未被实际更新。

- [ ] **2.1 更新已见牌信息**
  - 把本局已打出的牌、自己手牌、明刻/明杠从牌山中扣除。
  - 修复 `algo.py` 中 `TilePool.consume()` 的 `del self.pool` bug（应为 `del self.pool[t]`）。

- [ ] **2.2 基于弃牌修正对手手牌分布**
  - 某玩家连续打出某花色 → 该花色剩余牌可能更多在他家外。
  - 某玩家久未打出某花色 → 该花色可能在其手牌中偏少。
  - 用简单贝叶斯或计数器实现，不求精确，但求比均匀强。

- [ ] **2.3 危险牌概率加权**
  - 对每张待打牌，估计各对手已听牌且等待该牌的概率。
  - 前向搜索目标中加入点炮风险：
    ```python
    value(打出 t) = 进攻价值(t) - λ * Σ 对手胡牌概率(打 t)
    ```

## 三、搜索深度与效率

- [ ] **3.1 增加前向深度**
  - 当前默认 `eval2`（2-ply），可尝试 `eval3`、`eval4`。
  - 评估性能瓶颈，必要时用缓存/剪枝控制耗时。

- [ ] **3.2 候选动作剪枝**
  - 14 张牌的去重候选最多 13 种，可先快速过滤明显废牌。
  - 只保留 top-k 进入深层期望搜索。

- [ ] **3.3 蒙特卡洛 Rollout**
  - 用随机摸牌序列模拟到牌局结束，统计胜率/期望得分。
  - 从“精确枚举期望”向“采样期望”扩展，适合 deeper lookahead。

- [ ] **3.4 缓存与并行**
  - 对 `eval_seq`、`eval_suit` 等加 LRU cache。
  - 候选打法评估相互独立，可并行化。

## 四、动作空间扩展

当前 AI 只会摸牌→打牌，不会吃、碰、杠、报听。

- [ ] **4.1 碰/杠决策节点**
  - 当别人打牌时，增加分支：不碰 / 碰 / 杠。
  - 比较三种选择的期望价值，选择最优。
  - 杠之后要考虑额外摸牌（岭上开花）和改变牌山的副作用。

- [ ] **4.2 报听决策**
  - 晋北灵魂规则：报听后手牌锁定，摸非胡牌必须原样打出。
  - 如果当前已听牌，评估“报听” vs “不报听继续改良”的期望收益。
  - 报听分支的后续 forward sim 可大幅简化。

- [ ] **4.3 不能吃牌规则的利用**
  - 晋北多数局不允许吃，这减少了搜索分支，可显式约束动作空间。
