# 项目评审（Fable 5，2026-07-06）

> 本文基于对项目全部 23 篇 Markdown 文档的通读（AGENTS.md、README.md、docs/handoff.md、docs/designs/*、docs/reports/*、docs/rules.md、docs/expectimax-todos.md、output/ 下报告），不涉及代码阅读与实验运行。目的：评价做得好与不好的地方，并给出下一步最有潜力的方向。

---

## 一、做得好的地方

### 1. 阴性结果被诚实、系统地记录 —— 这是本项目最大的资产

绝大多数研究型项目只留成功记录，本项目把失败写得比成功还详细：DPO、KTO、PPO（至少 4 轮变体）、AWBC、TD(λ)、oracle 蒸馏、perfect-info rollout oracle、212 维危险特征、错题本蒸馏、赢家蒸馏、128k 数据缩放、AlphaZero 三轮 bootstrap、对手建模两种接入方式……每一项都有配置、数字和根因分析。这份「已证伪清单」（handoff §7）能让任何后续协作者避免重复至少数十小时的无效计算，其价值不亚于当前 best 模型本身。

### 2. 根因分析质量高，形成了可迁移的「项目定理」

几条贯穿多个实验、被反复验证的洞见值得点名：

- **「搜索强度上限 = leaf value 质量」**（rl-ppo-report §9）：一句话解释了为什么 depth-2 教师、DetMCTS、MCTS trace value 全部失败。
- **「教师必须本身在目标分布上强」**（conv-bc-roadmap 方向 9）：解释了为什么纯 BeliefExp 教师的 trace 蒸馏成功、而 V3-eval0 教师失败。
- **「纯自对弈坍缩到消极引分均衡」**（draw=0 ≻ loss=−1）：对稀疏奖励自对弈的精准诊断。
- **「TD value 与 residual leaf 公式语义冲突」**（td-lambda-plan §9.7）：value 预测更准但 Elo 更低的反直觉现象被解释清楚了。
- **「改输入失败、改目标函数成功」**（危险特征 vs deal-in auxiliary loss 的对照）：这是特征工程 vs 目标函数工程的经典教学案例。

这些结论不是事后追认，而是通过对照实验（如 v3nnpck:10 控制「候选更多」的混杂因素）主动拆分出来的，方法论上相当扎实。

### 3. 消融研究与「减法」意识

`ablation_report.md` 是全项目质量最高的单篇文档：固定 anchor、逐个移除组件、量化每项贡献（BeliefExp 搜索 +32pp、response head +22pp、dealin head ≈0），最后给出「哪些可以删」的清单。多数项目只做加法，这里主动做减法并敢于承认「128k 训练、对手建模、DPO/PPO/KTO 都可以删」。

### 4. 工程纪律成文且来自真实教训

AGENTS.md §9「血泪教训」（永不 kill 长任务、checkpoint 强制、清理前列出确认、小数据先验证、沉没成本优先）是从实际事故中提炼的操作守则，且后续文档能看到它被执行（AZ 数据生成加了 checkpoint/resume，HPO 脚本有日志汇总）。`lru_cache(maxsize=...)` 防 PyPy OOM、`torch.set_num_threads(1)` 防线程超订、DataCollector 决策前快照等坑都被记录成了 checklist。

### 5. 对评测陷阱有明确认知

项目自己发现并记录了三个关键评测陷阱：Elo 跨 run 漂移（必须同 pool）、vs-frozen 胜率不代表真实强度、小样本 Elo 不可信应以胜率为准。`benchmark_pool.py` 的同 pool 设计就是对这些认知的落实。

### 6. 确实取得了实质进展

从 Baseline（Elo ~1350）到 Hybrid-FullAction-SoupDistilled（同 pool Elo 1629、胜率 34.2% vs Baseline 17.8%），路径清晰可复盘：conv 架构 + BC → Hybrid 分层 → search trace distillation → 完整动作空间 → model soup + 蒸馏。其中 model soup（零训练成本）和「soup 当教师蒸馏回单一模型」是在多方向枯竭后找到的干净突破，性价比极高。

### 7. 文档体系有分层设计

AGENTS.md（环境+守则）→ handoff.md（状态+下一步）→ docs/README.md（索引）→ designs/（事前设计）→ reports/（事后报告）的结构是对的，且 docs/README 明确了「报告以完成时结论为准、过时追加更新段落而非覆盖」的归档规则。

## 二、做得不好的地方

### 1. 统计功效不足是全项目最系统性的问题

这是最需要直说的一点：**项目的大量晋升/放弃决策建立在噪声级差异上**。

- 4 人局胜率基线 25%，400 局单 agent 胜率的标准误约 ±2.2pp，95% CI 宽度接近 ±4.3pp。而项目里大量结论的效应量恰恰在 1–3pp：Soup2 比 base 高 1.2pp（1600 局）、AWBC v1/v3「略好」、hpo_0 混合池好但头对头差 1.5pp……这些全部在噪声范围内。
- 文档自己也多次记录了矛盾证据：hpo_0 在 400 局领先 7.7pp、1600 局缩到 0.4pp、头对头反而落后；BE16k_t8 与 BE8k_t8 在 2000 局确认中胜率仅差 0.1pp（25.8% vs 25.7%）却被认定为升级。
- **多重比较问题从未被处理**：项目累计测过几十个候选，每次挑最好的晋升。在噪声 ±2pp、候选数十个的情况下，「当前 best」有相当比例的优势可能是赢家诅咒（winner's curse）挑出来的运气。第二轮 bootstrap「无法继续提升」的一个未被讨论的解释是：上一轮的提升本身有一部分是噪声。

**建议的判断标准**：任何晋升决策，效应量至少要 2 倍于标准误，且必须在独立复跑中复现。按此标准回看，soup→蒸馏链条中真正过硬的可能只有「Hybrid vs 纯 NN」「full-action vs conv-BC」「conv-BC vs 小 MLP」这几个 20-30pp 级的大效应。

### 2. Elo 被反复使用，尽管项目自己已证明它不可靠

rl-ppo-report §4b 明确写了「此构成下 Elo 自相矛盾、不可信，以胜率为准」，但之后的文档（handoff §6.11–6.14、AGENTS.md §4）仍在用同 pool Elo 差作为主要证据，甚至出现胜率与 Elo 严重背离的表格照登不误（§6.12 800 局：Hybrid-base 胜率 24.8% 却 Elo 1286，BeliefExp 胜率 18.2% 却 Elo 1591——这组数字本身就说明该 Elo 实现对座位/对局顺序敏感到失真）。既然已知 pairwise Elo 在 4 人非对称池中漂移，就应该从报告模板里删掉它，或修好它（如 BradleyTerry 拟合全部对局），而不是「知道有毒但继续吃」。

### 3. 评测协议不统一，跨实验数字不可比

每个实验的 pool 构成、局数（80–2000）、对手组合都不同，导致：同一个 agent 的胜率在不同报告里从 17% 到 47% 都出现过；「当前 best 是谁」需要读者自己拼线索。没有一个固定的「标准考场」（固定对手、固定局数、固定席位轮换、固定种子集），这让 8 轮 search-distill 实验的纵向对比全靠间接推断。

### 4. 文档冗余、漂移与相互矛盾

- **「当前最强」在不同文档里不一致**：README.md 说是 Hybrid-BE16k_t8（Elo 1581），handoff.md/AGENTS.md 说是 Hybrid-FullAction-SoupDistilled（Elo 1629）。AGENTS.md 内部也自相矛盾：§1 说最强是 `BeliefExpectimaxV3Agent`（V3-NN-PC），§4 说是 SoupDistilled。
- **同一结果在 4 处重复**（handoff、AGENTS、conv-bc-roadmap、rl-ppo-report 各存一份 Hybrid/soup 结果），每次更新 best 需要同步 4 处，实际并没有同步到位。
- handoff.md 已膨胀到 47KB、§5 与 §6 分属两个时代（换机前 V3-NN-PC 时代 vs 换机后 conv 时代），Elo 1581 在两个时代含义完全不同（一个是 V3-NN-PC 的、一个是 BE16k_t8 的），极易误读。
- 小错误：备份文件命名 `nn_full_action_best_2024-07-05.pt`（应为 2026）；docs/README 说 recent-work.md「已合并/删除」但该文件仍存在并被索引。

### 5. 「胜率 or Elo or 点炮率」的目标函数问题被提出后从未回答

mahjong-ai-research-designs §8 明确问了「我们以什么为目标？」——这个问题直接决定 reward shaping、报听阈值、防守权重的一切取舍，但四个月来没有任何文档给出答案。结果是评价标准在文档间漂移：有时以胜率为准（PPO 报告），有时以 Elo 为准（soup 晋升），有时以点炮率为亮点（dealin head）。「更守但不更强」的结果（点炮 18%→15%、胜率不变）到底算不算改进，至今没有裁决标准。

### 6. 部分放弃决策的样本量配不上决策的重要性

80 局验证就否决了「depth-2 教师蒸馏」整条路线（§6.2）；100 局否决 MCTS-value leaf；shanten-ukeire 实验自己注明「流局率 82%、有效对局仅 18 局」但仍然给出了方向性结论。相比之下，晋升 best 用了 1600–2000 局。**否决一条大方向所需要的证据强度不应低于晋升一个模型**——尤其 depth-2 教师这种理论上限最高的方向，80 局（每 agent ~20 次胜负事件）远不足以判死刑。

### 7. 重复试错同一类已证伪的方法

PPO 从 2026-07 初到中旬被尝试了至少 4 轮（MLP 自对弈、对手式、conv-BC 热启、128k 热启、BE16k_t8 热启），每轮结论都是「从强 BC 初始化退化」。第 2 轮之后的重试没有引入新的关键变量（如 KL-to-reference 正则、更小 lr 的 LoRA 式微调），属于对同一假设的重复检验。类似地，outcome-level 的偏好学习在 DPO 失败后又用 KTO 重试了一遍，而两者共享同一个已被诊断的死因（样本级 ±1 reward 不是动作级信号）——这个死因在 DPO 失败时就可以推出。

### 8. 数据管理隐患

handoff §2 记录把多个 `.npz` 训练数据强制加入 git（最大 68529 条样本级别），而 16GB 的 128k 数据与大量 checkpoint 只存在本地且 `output/` 被 gitignore。当前 best 的完整复现链（教师数据 → soup 成分 → 蒸馏数据）依赖多个未纳管的本地大文件，换机时已经真实地丢过一次 best_1581 备份和 5000 局数据（rl-ppo-report §1）。教训发生过一次，但备份策略仍未成文。

## 三、下一步建议的有潜力方向

按优先级排序。核心判断：**项目当前是「评测噪声受限」而非「想法枯竭受限」**——在把测量精度修好之前，继续堆算法实验的边际收益会持续走低。

### 方向 1（最高优先级）：复式赛制评测，一次性把测量精度提高 3–10 倍

这是全项目 ROI 最高、且从未被尝试的杠杆。麻将方差极大，但其中绝大部分来自发牌运气，而运气是可以配对消除的：

- **复式（duplicate）设计**：同一副牌墙 / 同一随机种子下，让待比较的两个 agent 轮换坐同一席位各打一次，其余三席固定。比较的是「同一副牌下的成对差异」，运气项被差分掉。桥牌/后端麻将 AI 评测（如 Suphx 论文）都用类似手段，方差缩减通常达 3–10 倍，等效于把 400 局变成 1200–4000 局。
- **固定标准考场**：固定对手三件套（Baseline / BeliefExp / 上代 best）、固定种子集、固定局数（如 1000 副 × 2 镜像），所有候选跑同一考场，输出「胜率差 ± 95% CI」而非 Elo。
- **晋升门槛成文**：CI 不含 0 才晋升；晋升前独立种子集复跑一次。

工程量：主要是 engine/tournament 支持指定种子与席位轮换，估计远小于任何一个已做过的算法实验。**做完这件事后，建议第一个动作是用新协议复核 SoupDistilled vs 32k-best vs BE16k_t8**——当前 best 链条上有多环是 1–2pp 级差异，值得重新确认，避免后续所有实验对着一个部分由噪声选出的 anchor 做优化。

### 方向 2：把搜索做便宜，而不是把 NN 做强

消融已证明 BeliefExp 搜索是最大正收益（+32pp），且「搜索强度 = leaf value 质量」“教师必须更强”两条定理共同指向一个结论：**唯一被验证能产生更强教师的方法是更多搜索算力**。而 depth-2 被放弃的原因是慢（5s/步），不是弱（80 局的否决样本远不充分，见 §二.6）。

- 把 `eval2`/BeliefExp 核心热路径用 Cython/Rust/C++ 重写（历史上去 deepcopy 已拿到 5.5×，说明热点集中、可压缩空间大）。若再拿到 5–10×，depth-2 教师的数据生成就从「小时级不可行」变成「与现有 trace 蒸馏同量级」。
- 这直接复活两条已被判「理论最优但工程不可行」的路线：depth-2 search trace distillation（更强教师）与 true perfect-info rollout oracle（rollout policy 太慢的问题同时被解决）。
- 相比之下，「更大网络 + SE/attention」（§6.14.2，进行中）是在教师质量不变的前提下扩学生容量，历史先验（128k 缩放阴性、大网络从零训练失败）明确偏负，建议完成当前预训练后如无突破即止损。

### 方向 3：利用晋北规则的独特结构 —— 报听后的精确终局求解

这是文档里从未出现过、但与规则高度匹配的方向。晋北麻将「报听锁手」意味着：一旦某家报听，其后续行为完全确定（摸啥不胡就打啥）。因此终盘的博弈树会急剧收缩：

- 对手报听 + 牌山剩余不多时，「每张弃牌的点炮概率」和「自摸/放铳的期望收益」可以在 belief 分布上**精确计算**（对报听者的待牌集合做贝叶斯枚举），不需要 NN 也不需要 rollout。
- 这恰好覆盖 Hybrid 切换 BeliefExp 的同一批 critical 状态——即当前架构中价值最高的决策点。把 BeliefExp 在这些状态换成精确求解器，得到一个**严格更强的教师**（至少在终盘），再蒸馏。
- 项目多次失败的「防守学不会」问题（safety oracle、危险特征、加权 BC 全阴性），根源都是防守标签不准或不稳；精确终局 EV 是第一个真正意义上的 ground-truth 防守标签。

### 方向 4：对手建模换目标——从「是否听牌」改为「待牌分布」

对手建模第一轮失败的直接原因写得很清楚：二分类 acc 0.840 vs 常数基线 0.826，信号只有 1.5pp。但这不说明对手建模没戏，只说明**目标太粗**。听牌与否只有 1 bit；防守真正需要的是「对手待哪些牌」的 34 维分布。自对弈数据里对手手牌是免费的完美标签，per-tile 待牌预测的监督信号密度是二分类的几十倍。产出可直接接入两处：belief 更新（BeliefExp 的 per-player 信念当前仍是启发式）和方向 3 的终局求解的先验。

### 方向 5：外部数据 / 断开自我蒸馏循环

「引入人类/高水平对局数据」在 handoff §7 里挂了很久但从未执行。当前所有教师（eval2、BeliefExp、Hybrid 自身）同源，自我蒸馏已两轮收敛，这是典型的分布内饱和。如果晋北麻将确实拿不到人类牌谱，退而求其次的选项是**规则近邻迁移**：推倒胡类麻将（如四川麻将去缺门限制的简化版）的公开数据/引擎做预训练再迁移，或至少用风格差异大的 agent 池（激进/保守/随机混合）生成更多样的对局分布，缓解教师同源问题。

### 不建议继续投入的方向（基于已有证据）

- 任何形式的 outcome-level 离线 RL（DPO/KTO/RWR 变体）：死因相同且已三次验证。
- 从强 BC 初始化的 vanilla PPO：五次同结果。
- 同架构、同教师下继续 soup/蒸馏 bootstrap：§6.13 已确认边际递减。
- 无新变量的超参搜索：§6.14.1 已示范其一轮收益 ≈ 0。

## 四、一句话总结

这是一个**实验诚实度和根因分析质量显著高于平均水平**的研究项目，四个月内把 Elo 从 ~1350 推到 ~1629，并沉淀了一份罕见完整的阴性结果库；它当前最大的短板不在想法而在**测量**——评测噪声（±2pp）已经大于近期所有候选改进的效应量（1–3pp），继续在这个信噪比下做实验等于用糊的尺子量毫米。先做复式赛制评测（方向 1），再用省下来的方差预算去攻「更便宜的搜索 → 更强的教师」（方向 2）和「报听终局精确求解」（方向 3），是当前证据下期望收益最高的路径。

## 更新（2026-07-16）：方向 1 的「best 存疑」依据被证伪

本文方向 1 与 §一/§二多处引用「duplicate 考场 Baseline 显著强于 best（−20.2%）」作为
「best 链条部分由噪声选出」的证据。**该数字是 `benchmark_duplicate.py` 旧版配对统计
bug（候选与对手同名时前缀匹配误计）的产物**。用席位识别重算全部历史 pkl 后：

- Hybrid-Best − Baseline = **+9.4% [+8.0,+10.9]**（5000 pairs）/ +7.0% [+3.9,+10.1]（1000 arena）；
- Hybrid-Best − BeliefExp = **+10.4% [+9.0,+11.8]**（5000 pairs）/ +7.1% [+4.1,+10.1]（1000 arena）。

「当前 best 不如 Baseline」的危机判断**不成立**。但 winner's curse 在 soup→蒸馏这一环
得到证实：NewBest − OldBest 5000 pairs = +0.2% [−0.5,+0.9]、score-proxy +0.003 [−0.013,+0.020]，
两模型同强。本文的系统性批评（统计功效、多重比较、协议不统一、目标函数未定）不受影响，
仍有效；评测协议已据此成文：`docs/eval-protocol.md`。详见
`docs/reports/duplicate-reanalysis-0716.md`。
