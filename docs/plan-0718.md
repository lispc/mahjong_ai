# Plan 0718：简化路径与进一步提升方向（含执行计划）

> 2026-07-18。背景：`docs/reports/jax-rl-0717.md`（方向 1b 晋升 + 二代收敛判死）。
> 本文记录 (2) 维持性能下的简化结论、(3) 进一步提升的方向评估与当前执行计划。

## 1. 简化结论（维持性能）

### 1.1 部署侧（空间小）

- **搜索层不可砍**：纯 NN 独立使用实测胜率 7.5%（搜索层贡献 ~32pp，消融铁证），
  Hybrid 双组件架构必须保留。
- 可挤的小水（合计 ~1-2% 速度 + 整洁度，无更多）：
  - tenpai_threshold 28→32（阈值扫描噪声内无差，搜索触发略少）；
  - 剪 dealin head（Hybrid 内无贡献，消融已证）。

### 1.2 训练侧（空间最大）

- 当前 best 血统链（eval2→BeliefExp→BC→soup→蒸馏→gumbel）可压缩为两步：
  **任意 BC 级网络 + gumbel 闭环 12M**。
- 剩余唯一拐杖 = BC 初始化。gumbel 目标是**密集**的（不像 outcome PPO 冷启动
  必坍缩），**from-scratch 的 gumbel 训练第一次有可行性**（随机初始化 Q 虽噪声，
  但 legal mask + Gumbel 探索 + 每步密集标签可能自举）——未验证，见 §2.4。

### 1.3 架构侧

- dealin 头是死重可剪；Hybrid 双组件是架构疣，但「纯 NN 达搜索强度」的蒸馏
  本项目失败 N 次、gen2 自蒸馏饱和——**短期不动架构**。

## 2. 进一步提升：方向评估（按预期价值排序）

### 2.1 更深/更宽搜索目标（首选，执行中）

gumbel 管线唯一未动的核心旋钮：当前 1-ply + V 截断 + draws=2。
升级到 best-first 2-ply（top-2 候选展开第二层），同时修掉两个 v1 近似
（跳过报听决策点、跳过中间对手行动的部分）。成本 ~3× 搜索开销，
吞吐 911→~300 dec/s（12M ≈ 11h，可接受）。

### 2.2 对局多样性 / 联盟（治 gen2 死因，执行中）

gen2 死于自蒸馏回声室。引入对手池打破自引用并贴近 arena 分布：
- 80/20 混合（当前代 + 历史代冻结快照）；
- **异质对手**：JAX 实现 shanten 贪心 agent（规则层已有 `shanten_counts`，
  手牌效率风格，与 Baseline(eval2) 风格不同即可达多样性目的；
  Baseline eval2 的 JAX 移植工程量大且难向量化，不做）。

### 2.3 更大容量（候选）

BC 时代 128ch 到顶（128k plateau），但 12M 搜索标签丰富得多，
256ch/8blocks 可能吃得下。便宜，作为 2.1/2.2 之后的第三批候选。

### 2.4 from-scratch gumbel 训练（研究价值 > 强度价值）

验证「规则进、策略出、无教师」的优雅路径（§1.2）。KL 锚改为上一代自己
（NPG 式移动锚）或取消锚但保留 entropy 课程；用 gen1 管线的全部设施，
仅换 init=随机。冷启动风险：早期 Q 全噪声，学习可能极慢。

### 2.5 其他（低优先）

- value 目标换搜索值（AZ 标准）；
- arena 语义对齐（七对子/副露 quirk，卫生项）；
- 真实计分接入（方向 5，产品决策，可复活报听类结论）；
- 奖励整形（guardrail 点炮 +2.3pp，弱杠杆，可能丢胜率）。

## 3. 执行计划（2026-07-18 启动）

| 项 | 内容 | 工程 | 训练 | 裁决 |
|---|---|---|---|---|
| A | best-first 2-ply 搜索目标 | search.py 扩展（~1 天） | 12M（from gen1，anchor=gen1） | 协议全流程 |
| B | 对手池（current+gen1+BC+shanten-greedy） | ppo.py 对手池 + greedy agent（~1 天） | 12M（同 A） | 同上 |

**预登记门**：
- 工程门：A 的吞吐 ≥300 dec/s（否则退 k=8/draws=2 的加宽版：draws 2→4）；
  B 的 greedy agent 与 Python 参考实现分布一致。
- 训练门：KL 稳定不发散、无坍缩（流局 <10%）。
- 筛查门（两候选一批，多重比较控制：最多晋升 1 个，取点估计高者进确认）：
  1000-pair vs gen1 ≥ +1.0% → 5000-pair 独立种子确认 → 复跑符号一致 → 晋升。
- 若 A、B 均有效：合并配置（2-ply + 对手池）作为第三批候选，同流程裁决。
